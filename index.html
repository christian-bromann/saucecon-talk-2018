Reporters Testrunner Configuration Management Assertion Extendability Debugging CLI Setup & Teardown Integration Watch Mode Test Hooks
  • Reporters: You want to have nice spec or junit reports that actually specify the capabilities and other meta information of the test it is reporting
  • Testrunner: You want a tool that runs your tests in parallel for you to speed up the test execution time
  • Configuration Management: This is about seperation of concerns: you want to have the configuration aspects of your tests seperated from your test
  • Assertion: It is easier to identify errors with error messages and snapshot comparisons as well as it makes the code more readable
  • Extendability: You wanna be able to extend your suite with plugins to solve problems other people have already solved
  • Debugging: This is the most important feature of all. As easier it is to debug your automated test as easier it is for you to build stable test suites
  • CLI: A nice command line interface can not only help to set up things quicker but also to easier run test suites on CI
  • Setup & Teardown: Let the test lifecycle be managed by a tool instead of yourself
  • Integration: with IDEs (e.g. code completion) or arbitrary tools (e.g. Grunt, Gulp)
  • Watch Mode: Be able to rerun your test suite everytime you hit safe (not that important)
  • Test Hooks: Execute certain tasks when certain events appear (e.g. take screenshot when test fails)

  • In WebDriver you have
  • 1) your test script using a test framework (could be written in any language)
  • 2) a WebDriver endpoint (a browser driver)
  • they communicate of the WebDriver specification (which is not more or less a rule set)
  • 3) at the end is the browser
  • The communication between browser (target device) and driver depends on the target device
  • For mobile it is even a bit more complicated
  • If you do web testing on the phone Appium is also using Chromedriver for example
  • But before it relays the commands to it it uses ADB or XCode to setup the device for you
  • For testing native Apps Appium automatically switches to the OS's native automation framework
  • At the end all you need to do is speak webdriver like in Web
  • I get a lot of questions that are like: "Does WebdriverIO support IE11?"

WebDriver vs Cypress

  • One protocol to rule them all
  • Highlevel access to browser functionality (open, click, get text)
  • Usable with all programming languages
  • Limited to Chrome based browser (Firefox support in the making)
  • In depth access to browser functionality (request mocks, DOM snapshots)
  • Only support for Node.JS
  • The advantage of having something like a WebDriver specification allows us to write driver for literally every end device really easy and straightforward
  • This is why Appiums has the vision to become the StarDriver
  • The Problem: the WebDriver spec is the lowest denominator for commands that can be supported by all these devices
  • It lacks on features like Request Stubbing or other debugging utilities (e.g. DOM snapshots)
  • However: I am super confident that the people working on that specification will add more useful commands or features that makes our e2e tests better
  • Just a hint: in the last talk with Slav and myself we will introduce extended debugging at sauce so you might want to check that out
  • Cypress.io has currently not the problem of having to support so many devices

import WebDriver from 'webdriver'
 
;(async function () {
    const client = await WebDriver.newSession({
        path: '/',
        capabilities: { browserName: 'firefox' }
    })

    await client.navigateTo('https://www.google.com/ncr')

    const searchInput = await client.findElement('css selector', '#lst-ib')
    await client.elementSendKeys(searchInput['element-6066-11e4-a52e-4f735466cecf'], 'WebDriver')

    const searchBtn = await client.findElement('css selector', 'input[value="Google Search"]')
    await client.elementClick(searchBtn['element-6066-11e4-a52e-4f735466cecf'])

    console.log(await client.getTitle()) // outputs "WebDriver - Google Search"

    await client.deleteSession()
})()

import { remote } from 'webdriverio'

;(async function () {
    const client = remote({
        path: '/',
        capabilities: { browserName: 'firefox' }
    })

    await client.url('https://www.google.com/ncr')

    const searchInput = await client.element('#lst-ib')
    await searchInput.setValue()

    const searchBtn = await client.element('input[value="Google Search"]')
    await client.click()

    console.log(await client.getTitle()) // outputs "WebDriver - Google Search"
    await client.deleteSession()
})()
